# 智能归档：背后的技术思考

## 视觉与语义的鸿沟
在传统的素材整理中，我们面临两个极端：
1.  **纯数字/日期命名**: 机器友好，但人类无法阅读（如 `IMG_1284.mp4`）。
2.  **纯人工打标**: 语义丰富，但成本极高。

本项目通过以下三个核心环节，在“自动化”与“高语义”之间找到了平衡。

## 技术方案解析

### 1. 聚类：让相似的走在一起
我们使用了 **HDBSCAN** 算法。与常见的 K-Means 不同，它能够：
*   **处理杂质**: 如果一个视频跟谁都不像，它会被归类为 `Others`。
*   **无需预设分类数**: 它能自动发现你的素材库里有几类内容。

### 2. 代表素材提取：效率的关键
系统不是在分析这 22 个视频，而是在分析 **6 个中心点**。
*   **逻辑**: 每一个聚类簇，我们只挑选最靠近几何中心（Centroid）的那一个视频作为“代表”发送给 Gemini 3.1。
*   **优势**: 极大降低了 API 消耗，同时保证了命名的代表性。

### 3. 多维命名逻辑
我们强制模型输出 `主体_场景_物件_氛围` 组合标签。

| 优化前 | 优化后 (当前方案) |
| :--- | :--- |
| 小女孩 | 女孩_公园_泡泡棒_玩耍 |
| 职场女性 | 职场女性_办公空间_笔记本电脑_高效商务 |
| 电脑主机 | 电脑主机_工作室_UGREEN机箱_科技感 |

## 代理处理与优化
*   **720p 镜像**: 脚本会自动优先处理低分辨率副本，平衡画质分析需求与网络传输带宽。
*   **智能兜底**: 如果文件名本身包含重要信息（如 `电量不足`），系统会引导 AI 将其融入最终标签。
