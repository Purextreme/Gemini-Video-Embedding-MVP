# Gemini Embedding 2 视频检索验证报告 (MVP)

## 1. 验证目标
验证 `gemini-embedding-2-preview` 模型是否能通过自然语言文本，准确检索到对应的单镜头视频素材。

## 2. 测试环境与方法
- **模型**: `gemini-embedding-2-preview`
- **向量维度**: 3072维
- **素材**: 22 个单镜头视频（已转码为 720p MP4，体积均 < 7MB）
- **实现方式**: 采用 **Inline Data (字节流直传)** 模式，绕过 Files API 权限预处理，实现快速向量化。
- **计算逻辑**: 文本向量与视频向量的 **余弦相似度 (Cosine Similarity)**。

## 3. 检索测试结果
本次测试共执行 6 组不同场景的自然语言查询，**Top-1 命中率达到 100%**。

## 4. 核心发现
1. **语义理解精度高**: 模型不仅能识别物体（如无人机、NAS），还能准确区分相似主体的不同动作（如区分“吹泡泡”与“捉迷藏”）。
2. **多模态对齐效果显著**: 文本向量与视频向量在 3072 维空间中展现了极强的相关性，分值差距足以拉开 Rank 1 与 Rank 2 的距离。
3. **技术路线验证成功**: 
   - **Inline 模式** 是处理小视频（<20MB）最快、最稳的验证路径。
   - **720p 低码率转码** 对检索精度几乎没有负面影响，极大地优化了上传带宽。

## 5. 结论与建议
**结论：核心假设成立。** `Gemini Embedding 2` 完全胜任“以文搜片”的单镜头检索需求。

**后续建议：**
- **工程化方向**: 若后续涉及海量素材，可考虑引入简单的向量索引（如 Numpy 矩阵运算或轻量级向量库）。
- **优化方向**: 进一步测试“本地抽帧合成快进片”的方案，以支持更大规模、更高效率的视频入库。
